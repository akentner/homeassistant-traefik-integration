     except (TimeoutError, asyncio.TimeoutError) as exc: return {"host":
     host, "port": port, "error": "timeout", "detail": str(exc) or

<threat_model>
<security_enforcement>true</security_enforcement>

**Threat: TLS handshake to attacker-controlled hosts**
- Hostnames are derived from `entry.runtime_data.data["http_routers"][i].tls.domains[]`
  — these are values the user controls via their Traefik configuration but
  the integration doesn't validate them. An attacker who can modify Traefik
  config (e.g., a compromised container image) could inject a hostname that
  resolves to their malicious server.
- **Mitigation:** Every handshake uses stdlib `ssl.PROTOCOL_TLS_CLIENT` with
  `load_default_certs()` — chain validation is enforced. The cert is fetched
  but `check_hostname=False` is required to avoid false-positives on
  Traefik default certs (a deliberate trade-off documented in spike 005).
  `san_mismatch: bool` on `CertInfo` surfaces the case where the probe
  hostname isn't covered by the cert SAN (spike 006) — this is an
  attribute visible on the timestamp sensor (plan 03-02). If the probe
  fails on a hostname the attacker controls (e.g., drops the connection
  silently), the entity marks `unavailable` — never crashes.

**Threat: Token leak via error messages**
- The integration's bearer token never appears in `tls.py` — the module
  is fully decoupled from the API client (TLS uses raw socket + ssl,
  not aiohttp; no auth header is sent in a TLS handshake by design).
- `CertError.detail` strings may contain exception messages from the
  underlying SSL/socket layer (e.g., `"ssl.SSLError: certificate verify
  failed: unable to get local issuer certificate"`). These are NEVER the
  integration's bearer token — they're the cert's issuer info or host
  diagnostics. Logged at `_LOGGER.debug` only when throttled (24h per
  host) — never `_LOGGER.error` to avoid HA's notification surface.

**Threat: DoS via large router counts**
- `_async_update_data` runs once per 6h. Even a hostile config with
  10,000 routers only triggers one cycle.
- `asyncio.Semaphore(4)` bounds concurrent handshakes to 4 — caps
  bandwidth + fd usage per cycle.
- Each handshake has `asyncio.timeout(5)` so a hanging host can't
  stall the cycle indefinitely; worst case `len(hosts)/4 * 5s`
  budget (e.g., 200 hosts = 250s = ~4 min).
- The cache is per-entry, in-memory only — no persistence means restart
  resets the data; no quota to consume.

**Threat: SNI downgrades / hostname mismatch**
- Handshake uses `server_hostname=host` so SNI routing is preserved
  (per spike 003). Without this, an attacker controlling the SNI
  negotiation could redirect probes to a different cert.
- `san_mismatch: bool` from spike 006 surfaces cases where the cert's
  SAN doesn't cover the probed hostname — visible in the entity's
  `extra_state_attributes` so dashboards can alert on it.

**Threat: Parse failures**
- Every `parse_not_after` failure is caught and translated to
  `CertError(error="parse")` — never raises.
- Unexpected shapes fail gracefully (CONTEXT.md D-11: log throttled
  to once per host per 24h to avoid log-spam).

**Threat: IPv6 unreachable**
- `socket.create_connection((host, port))` handles `[host]:port`
  natively (spike 004 verified). IPv6 failures map to the same
  `CertError(error="oserror")` enum as IPv4 failures — no special
  handling required. No bypass risk.

**Threat: runtime_data shape migration**
- Per PITFALLS #6, `entry.runtime_data` is NOT migrated to a wrapper.
  The cert coordinator is attached as a sibling attribute
  (`coordinator.cert_coordinator = cert_coordinator`) on the main
  `TraefikCoordinator`. This avoids breaking every existing
  `entry.runtime_data.X` access in the codebase (the reload service
  handler at `__init__.py:65`, the options listener, etc.). Plugin
  authors integrating with the entry can read either
  `entry.runtime_data` (main coordinator) or
  `entry.runtime_data.cert_coordinator` (TLS side).

**Threat: Probe logs leak hostnames to integration logs**
- `_LOGGER.debug` includes only `host` and `raw` strings — neither
  contains bearer tokens. The detail string in `CertError` is the
  exception message, which is sanitised by stdlib (no path or
  credential leakage on standard Python `socket` / `ssl` errors).
