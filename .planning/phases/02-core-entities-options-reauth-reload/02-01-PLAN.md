<objective>
Migrate the Phase 1 single-device entity model into a per-category multi-device model, extend the Traefik API client with the new endpoints (entrypoints, services, middlewares, overview) and the reload trigger, and update the coordinator's TraefikData shape so Phase 2 platforms can read the full API surface.

Purpose: Phase 1 ships one device and only /api/version + /api/http/routers. Phase 2 introduces five new platforms (per-entrypoint sensor, per-service sensor, three aggregate sensors, reload button, any-failing binary sensor) that need richer data and clearer device grouping. Without this plan, downstream plans 02-02..02-04 cannot instantiate the right entities on the right devices.

Output:
- TraefikEntity takes `category: str`; DeviceInfo derives identifier, model, name from category
- TraefikApiClient exposes 4 new GET endpoints + POST reload_routers + filter_internal_items helper
- TraefikCoordinator.fetch_all() returns TypedDict-shaped data covering all 6 endpoints
- Phase 1 binary_sensor continues to work unchanged after migration (entity_id prefix preserved)
- const.py exposes CONF_TLS_WARN_DAYS, MIN/MAX scan/ TLS clamps, and PLATFORMS extended to include sensor + button
</objective>
