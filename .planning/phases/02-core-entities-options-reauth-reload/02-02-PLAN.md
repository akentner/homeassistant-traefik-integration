<objective>
Extend the Phase 1 config flow with three lifecycle flows — Options (re-configure without remove+re-add), Reauth (token rotation), Reconfigure (URL change) — and wire an update listener that applies scan_interval changes live while triggering a full reload only when the URL changes.

Purpose: Phase 1 ships only `async_step_user` and `async_step_yaml`. Users have no way to change scan_interval, verify_ssl, or rotate the token without removing the integration entry. Phase 2 success criterion #1 and #2 (ROADMAP) require all three flows to exist and respect HA's standard lifecycle hooks.

Output:
- TraefikOptionsFlow (bound via ConfigFlow.async_get_options_flow) with scan_interval (15..300s), verify_ssl (bool), tls_warn_days (1..90) — clamps enforced via voluptuous schema
- async_step_reauth: same UX as gatus' reauth flow; password selector for new token; validates via /api/overview probe; updates entry.data in place; aborts with `reauth_successful`
- async_step_reconfigure: pre-fills URL + token from entry.data; validates via probe; calls async_update_reload_and_abort with data_updates
- __init__.py registers `entry.add_update_listener(_async_options_updated)` that mutates `coordinator.update_interval` when only options change, and lets HA's reload-on-entry-data-change handle URL rotation
- strings.json + translations/en.json extended with all new keys
</objective>
