# Spike Conventions

Patterns and stack choices established across spike sessions for this
project. New spikes follow these unless the question requires otherwise.

## Stack

- **Python 3.14** (matches `.python-version` + `pyproject.toml`).
- **stdlib-only** for spike code that mirrors production code (e.g.
  the TLS handshake prototype shares modules with what will land in
  `custom_components/traefik/`). No `cryptography` import — project
  mandates `manifest.json` `requirements: []`.
- **uv** for Python dep management; matches user's `~/.local/bin/uv`.
- **pytest + pytest-asyncio** for any persistent tests that survive
  into the integration codebase (spike scripts are throwaway; tests
  that ship live in `tests/components/traefik/`).

## Structure

```
.planning/spikes/
├── MANIFEST.md                          # parent index (this dir)
├── CONVENTIONS.md                       # parent meta-conventions
├── WRAP-UP-SUMMARY.md                   # generated by /gsd:spike --wrap-up
└── <NNN-spike-name>/                    # one dir per session (spike ID)
    ├── MANIFEST.md                      # session-specific index
    ├── CONVENTIONS.md                   # session-specific patterns
    ├── shared/                          # code reused across the session's spikes
    ├── <NNN>-descriptive-name/          # individual spike
    │   ├── README.md
    │   ├── probe.py                     # standalone runnable
    │   └── ...
    └── run-all.sh                       # single-command re-runner
```

After wrap-up, all spike artifacts also live at
`.claude/skills/spike-findings-<project>/` for auto-loading.

## Patterns

- **One `probe.py` per spike** that exits 0 on pass, 1 on fail.
  Lets the spike be CI-runnable. Spikes that need a server co-locate
  the server in the same spike dir.
- **`shared/` for cross-spike code.** E.g. `shared/tls.py` was used by
  spikes 001 + 003 + 004. The shared prototype becomes the production
  code (`custom_components/traefik/tls.py`) during plan execution.
- **Stdout-only observability** for fact-validation spikes (does
  library X work?). No forensic log layer needed for non-runtime spikes.
- **Markdown tables for results.** Each spike's `probe.py` prints
  `[PASS]/[FAIL]` lines that paste directly into the README.
- **Generate self-signed certs via `openssl` CLI** for TLS/SNI testing —
  avoids `cryptography` lib dependency, matches the stdlib-only mandate.
- **`asyncio.to_thread` wrapper** for any blocking stdlib call
  (`socket`, `ssl`) — event-loop responsiveness is non-negotiable.
- **Self-signed certs for testing: omit `basicConstraints=CA:FALSE`**
  if you need to load them as trusted CAs via `load_verify_locations`.

## Tools & Libraries

- **Python stdlib `ssl`** — `PROTOCOL_TLS_CLIENT`, `check_hostname=False`,
  `load_default_certs()`, `cert_time_to_seconds()`, `SSLContext.sni_callback`.
- **Python stdlib `socket`** — `create_connection((host, port))` works for
  IPv4 and IPv6 (`[::1]:port`).
- **Python stdlib `asyncio`** — `to_thread`, `timeout`, `Semaphore`,
  `start_server`, `open_connection`, `gather`.
- **`openssl` CLI** for cert generation (in spike 003's `generate.sh`).
- **`uv run`** for script execution in spike dirs.

## Anti-patterns to avoid

- `verify_mode=CERT_NONE` — returns empty `getpeercert()` dict.
- `load_cert_chain` inside `sni_callback` — off-by-one cert serving.
- `check_hostname=True` on the client probe — fails on Traefik default
  certs and wildcards.
- Forgetting `load_default_certs()` — `CERT_REQUIRED` fails without
  trusted CAs.
- Using `cryptography` library for cert parsing — adds manifest
  `requirements`; project mandates stdlib-only.
- `asyncio.get_event_loop()` — deprecated; use `asyncio.run()`.
- Per-entity async update instead of `DataUpdateCoordinator` —
  defeats the 6h cadence + semaphore pattern.
- Self-signed certs with `basicConstraints=CA:FALSE` for `load_verify_locations`
  test fixtures — they're rejected as non-CA roots.
